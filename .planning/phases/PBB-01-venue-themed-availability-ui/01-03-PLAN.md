<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User browser -> `web/app/s/[shareToken]/[venueId]` | Public URL includes share token and venue ID. |
| `web/` server component -> backend `/api/public` | Frontend fetches display-safe data from backend over HTTP. |
| Backend DTO -> React render | Text and URLs from cached availability are rendered in browser DOM. |
| `Open booking` link -> external venue site | User leaves the read-only page for the venue booking page. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-10 | Spoofing | `web/src/lib/publicAvailability.ts` | mitigate | Fetch only backend `/api/public`; do not trust client-derived venue metadata beyond display route params. |
| T-01-11 | Tampering | `DayCard` booking links | mitigate | Render backend-provided bookingUrl only as plain external links with `rel="noopener noreferrer"`; do not mutate booking state. |
| T-01-12 | Repudiation | web page read-only flow | accept | Page performs no writes or bookings, so no user action audit trail is required in Phase 1. |
| T-01-13 | Information Disclosure | `web` source and bundle | mitigate | No Supabase env vars, sync tokens, raw availability endpoints, extension storage keys, or scraping selectors in `web/`. |
| T-01-14 | Denial of Service | page fetch behavior | mitigate | Fetch once per request with `cache: "no-store"`; no polling, scraping, or refresh loops. |
| T-01-15 | Elevation of Privilege | public page actions | mitigate | No login, payment, checkout, waiver, CAPTCHA, access-control, or booking automation code. |
</threat_model>
