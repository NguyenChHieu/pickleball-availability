<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser or web app -> `/api/public/:shareToken/:venueId` | Untrusted public URL path includes a secret share token and venue ID. |
| Backend public route -> cache store | Route reads cached user-approved availability from local storage or Supabase through `availabilityStore`. |
| Cached payload -> public JSON | Internal cache data is transformed into allowlisted display fields. |
| Public JSON -> external booking URL | Day booking links leave the app and open the venue booking site. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | `validShareToken` in `server/src/index.js` | mitigate | Validate share token before any cache lookup; invalid token returns generic 404 per D-04. |
| T-01-02 | Information Disclosure | `server/src/publicAvailability.js` | mitigate | Build an explicit allowlist DTO; never return raw records, raw slot JSON, Supabase fields, sync tokens, or extension storage keys per D-02. |
| T-01-03 | Tampering | `bookingActionUrlForDay` | mitigate | Allow only http/https URLs, strip existing hashes, and append only encoded `#pbb_date=` markers. |
| T-01-04 | Repudiation | public endpoint failures | accept | Read-only display endpoint has no user write action; server keeps console errors for operator debugging without exposing them publicly. |
| T-01-05 | Denial of Service | public endpoint cache reads | accept | Endpoint performs one cache read and deterministic transformation; no scraping, polling, or external booking calls. |
| T-01-06 | Elevation of Privilege | route separation from `/api/availability` | mitigate | Do not reuse protected raw availability handlers; keep sync-token guarded endpoints unchanged. |
| T-01-SC | Tampering | npm installs | accept | No new backend packages are installed in this plan; server remains dependency-free CommonJS. |
</threat_model>
