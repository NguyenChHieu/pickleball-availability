<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser viewport -> visual UI | Untrusted client dimensions, reduced-motion settings, and browser behavior affect rendering. |
| Backend DTO -> polished DOM | Cached display data is styled and animated in the page. |
| User -> `Open booking` links | User leaves the app through external links. |
| Motion code -> availability content | Optional hero behavior could obscure or delay essential content if implemented poorly. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-16 | Spoofing | visual freshness copy | mitigate | Use `Last read` and stale warning copy; never imply live or guaranteed availability. |
| T-01-17 | Tampering | hero/motion layer | mitigate | Motion is decorative and cannot change DTO data, links, or booking state. |
| T-01-18 | Repudiation | external booking link clicks | accept | Links are read-only navigation; the app does not submit bookings or account actions. |
| T-01-19 | Information Disclosure | error/empty states | mitigate | State copy is display-safe and does not reveal raw backend errors, Supabase details, or token validity beyond generic not-found behavior. |
| T-01-20 | Denial of Service | client motion | mitigate | No continuous animation loops; reduced motion disables scroll reaction; no heavy canvas/3D packages. |
| T-01-21 | Elevation of Privilege | UI controls | mitigate | No in-page refresh, scraping, login, payment, checkout, waiver, CAPTCHA, or booking automation controls. |
| T-01-SC | Tampering | npm installs | mitigate | This plan adds no new direct packages; source gate forbids unplanned animation/canvas packages. |
</threat_model>
